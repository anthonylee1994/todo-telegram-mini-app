# API Doc for Telegram Mini App

This API is for the Telegram Mini App task UI.

All task endpoints require Telegram Web App authentication. The backend reads Telegram `initData`, verifies the Telegram hash with the bot token, then finds or creates the user from the embedded Telegram user payload.

## Base URL

Use the backend origin configured for the current environment.

Examples:

```txt
https://api.example.com
http://localhost:3000
```

## Authentication

Send the raw Telegram Mini App `initData` with every task API request.

Preferred header:

```http
X-Telegram-Init-Data: <window.Telegram.WebApp.initData>
```

Alternative header:

```http
Authorization: tma <window.Telegram.WebApp.initData>
```

Frontend example:

```ts
const initData = window.Telegram.WebApp.initData;

const response = await fetch(`${apiBaseUrl}/tasks`, {
  headers: {
    "X-Telegram-Init-Data": initData,
  },
});
```

If authentication fails, the API returns:

```http
401 Unauthorized
```

```json
{
  "error": "Unauthorized"
}
```

## Task Object

```json
{
  "id": "9d7f9f5f-8c9b-4b8f-9f79-8c6f5a4b2f17",
  "title": "Buy milk",
  "description": "Oat milk",
  "due_date": "2026-06-26T10:00:00.000Z",
  "status": "pending",
  "user_id": "2a0f5b7c-18d8-4c34-9a11-3d756a7b4a02",
  "created_at": "2026-06-26T02:00:00.000Z",
  "updated_at": "2026-06-26T02:00:00.000Z"
}
```

Fields:

| Field         | Type                        | Notes                                                   |
| ------------- | --------------------------- | ------------------------------------------------------- |
| `id`          | string                      | UUID                                                    |
| `title`       | string                      | Required                                                |
| `description` | string or null              | Optional                                                |
| `due_date`    | ISO datetime string or null | Optional                                                |
| `status`      | string                      | `pending` or `completed`                                |
| `user_id`     | string                      | Backend-owned. Do not send or trust this from frontend. |
| `created_at`  | ISO datetime string         | Generated by backend                                    |
| `updated_at`  | ISO datetime string         | Generated by backend                                    |

## List Tasks

```http
GET /tasks
```

Query params:

| Param     | Values                          | Notes                                                                           |
| --------- | ------------------------------- | ------------------------------------------------------------------------------- |
| `filter`  | `today`, `this_week`, `overdue` | Optional. Unknown values are ignored.                                           |
| `sort_by` | `due_date`                      | Optional. Only `due_date` is supported.                                         |
| `order`   | `asc`, `desc`                   | Optional. Defaults to `asc` when sorting by due date unless `desc` is provided. |

Example:

```http
GET /tasks?filter=overdue&sort_by=due_date&order=asc
```

Success:

```http
200 OK
```

```json
[
  {
    "id": "9d7f9f5f-8c9b-4b8f-9f79-8c6f5a4b2f17",
    "title": "Buy milk",
    "description": null,
    "due_date": "2026-06-26T10:00:00.000Z",
    "status": "pending",
    "user_id": "2a0f5b7c-18d8-4c34-9a11-3d756a7b4a02",
    "created_at": "2026-06-26T02:00:00.000Z",
    "updated_at": "2026-06-26T02:00:00.000Z"
  }
]
```

Notes:

- Only tasks owned by the authenticated Telegram user are returned.
- Passing `user_id` in query params has no effect.
- `overdue` returns tasks with `due_date` before the current server time and `status` not equal to `completed`.

## Create Task

```http
POST /tasks
Content-Type: application/json
```

Request body:

```json
{
  "task": {
    "title": "Buy milk",
    "description": "Oat milk",
    "due_date": "2026-06-26T10:00:00Z",
    "status": "pending"
  }
}
```

Allowed task fields:

| Field         | Required | Notes                                           |
| ------------- | -------- | ----------------------------------------------- |
| `title`       | Yes      | Must be present                                 |
| `description` | No       | Free text                                       |
| `due_date`    | No       | ISO datetime recommended                        |
| `status`      | No       | `pending` or `completed`; defaults to `pending` |

Success:

```http
201 Created
```

```json
{
  "id": "9d7f9f5f-8c9b-4b8f-9f79-8c6f5a4b2f17",
  "title": "Buy milk",
  "description": "Oat milk",
  "due_date": "2026-06-26T10:00:00.000Z",
  "status": "pending",
  "user_id": "2a0f5b7c-18d8-4c34-9a11-3d756a7b4a02",
  "created_at": "2026-06-26T02:00:00.000Z",
  "updated_at": "2026-06-26T02:00:00.000Z"
}
```

Validation error:

```http
422 Unprocessable Content
```

```json
{
  "errors": ["Title can't be blank"]
}
```

Notes:

- The backend always assigns the task to the authenticated Telegram user.
- Submitted `user_id` is ignored.

## Update Task

```http
PATCH /tasks/:id
Content-Type: application/json
```

Request body:

```json
{
  "task": {
    "title": "Buy oat milk",
    "description": "Unsweetened",
    "due_date": "2026-06-27T12:00:00Z",
    "status": "completed"
  }
}
```

All fields are optional, but at least one useful field should be provided.

Success:

```http
200 OK
```

```json
{
  "id": "9d7f9f5f-8c9b-4b8f-9f79-8c6f5a4b2f17",
  "title": "Buy oat milk",
  "description": "Unsweetened",
  "due_date": "2026-06-27T12:00:00.000Z",
  "status": "completed",
  "user_id": "2a0f5b7c-18d8-4c34-9a11-3d756a7b4a02",
  "created_at": "2026-06-26T02:00:00.000Z",
  "updated_at": "2026-06-26T03:00:00.000Z"
}
```

Validation error:

```http
422 Unprocessable Content
```

```json
{
  "errors": ["Status is not included in the list"]
}
```

## Frontend Fetch Helper

```ts
type ApiOptions = RequestInit & {
  json?: unknown;
};

export async function apiFetch(path: string, options: ApiOptions = {}) {
  const headers = new Headers(options.headers);
  headers.set("X-Telegram-Init-Data", window.Telegram.WebApp.initData);

  if (options.json !== undefined) {
    headers.set("Content-Type", "application/json");
  }

  const response = await fetch(`${import.meta.env.VITE_API_BASE_URL}${path}`, {
    ...options,
    headers,
    body:
      options.json === undefined ? options.body : JSON.stringify(options.json),
  });

  if (response.status === 204) {
    return null;
  }

  const data = await response.json();

  if (!response.ok) {
    throw new Error(
      data.error || data.errors?.join(", ") || "API request failed",
    );
  }

  return data;
}
```

Example usage:

```ts
const tasks = await apiFetch("/tasks?sort_by=due_date&order=asc");

const task = await apiFetch("/tasks", {
  method: "POST",
  json: {
    task: {
      title: "Buy milk",
      due_date: "2026-06-26T10:00:00Z",
    },
  },
});
```

## Non-Mini-App Endpoint

```http
POST /telegram/webhook
```

This endpoint is for Telegram Bot API webhook delivery, not for the Mini App frontend.

It uses:

```http
X-Telegram-Bot-Api-Secret-Token: <webhook secret>
```

Do not call this endpoint from the Mini App.
